# Execute Skill Commands Fresh

When a skill is invoked, execute every command in every step
as written. Never skip a command because prior results exist
in the conversation.

State changes constantly — flows start, issues close,
dependencies shift, labels move. Data from five minutes ago
is stale. The skill was invoked because the user wants
current state, not a replay of old state.

"I already have the results" is never a valid reason to skip
a skill's commands. If the skill says to run a script, run it.
